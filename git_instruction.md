# **Инструкция по работе с Git**

## Введение

Что такое *Git*? Зачем он нужен? *Git* - это система контроля версий. Или другими словами - хранилище, база данных истории разработки проекта. Такая система нужна для учета всех версий файлов когда-либо созданных в проекте и содержащихся в Git (базе данных). Если говорить совсем простым языком, то это программа, позволяющая сохранять прогресс в любой момент, а так же возвращаться к нему в любой момент, причём количестов таких сохранений, грубо говоря, не ограничено, так как программа сохраняет разницу версий, а не создаёт множество новых файлов. Благодаря этому экономит место на носителе информации.

## Установка программного обеспечения

Прежде чем начинать работу с *Git*, необходимо установить его на компьютер. Однако помимо этого потребуется некий программный редактор, который будет содержать в себе не только окно ввода и вывода данных, но и терминал. Для этого используется программа *Visual Studio Code*. Перейдя по ссылкам ниже, можно скачать вышеперечисленные программы. Их установка проста, просто следуйте инструкциям в диалоговых окнах.

Ссылки для скачивания:
* [Git](https://git-scm.com/)
* [VSC](https://code.visualstudio.com/)

## Настройка и проверка програмного обеспечения, регистрация в Git

### Настройка и регистрация в *Git*

После установки *Git* необходимо в нём зарегестрироваться. Это делается для того, чтобы другие пользователи, которым был отослан файл, могли видеть кто именно вносил корректировки и куда именно давать обратную связь. Чтобы это сделать необходимо:
1. Запустить программу *Git*
2. В открывшейся программе необходимо прописать следующие команды:
    * git config --global user.name «Ваше имя английскими буквами»
    * git config --global user.email ваша почта@example.com

Если этого не сделать сразу, то при первом же запуске программы, будет выдана ошибка, говорящая о том, что программа не знает пользователя и попросит пройти регистрацию, при этом работать не будет.

*Git* настроен и готов к работе. 

### Теперь необходимо настроить *Visual Studio Code*.


Автоматически интерфейс программы будет на английском языке, однако это можно исправить, установив руссификатор. Для этого необходимо перейти во вкладку ***extensions***, которая находится в первом левом столбце и выглядит как три квадрата сложенные в уголок, к которым подходит такой же по размеру четвёртый квадрат. Далее необходимо ввести в поиск ***Russian Language Pack***. Далее нажать клавишу ***instal***. После внизу справа выйдет диалоговое окно, которое будет предлогать перезагрузить программу уже с установленным интерфейсом на русском языке.

Однако это необязательный шаг, который можно пропустить. 

## Создание нового документа в VS Code

Для создания документа в программе необходимо выполнить следующие шаги:
1. Запустить *VS Code*
2. Нажать кнопку *Файл*
3. Нажать *Открыть папку*
4. В открывшемся диалоговом окне выбрать директорию, в которой будет создан файл.

После выполнения алгоритма выше, во втором столбце программы, который называется ***проводник*** появится директория, которую вы выбрали для работы. Далее создаём новый файл. Наводим на название вашей папки курсором и появляется несколько иконок, но нам необходима только одна: *Создать файл...* - она выглядит как лист бумаги со значком + в правом нижнем углу. Далее необходимо дать ему имя. Делать это рекомендуется латинскими буквами и обязательно с указанием расширения файла, например: ***git_instruction.md***, где "git_instruction" - название файла, а ".md" - его расширение.

Файл создан, можно приступать к работе с *Git*

## Создание терминала

Для работы с *Git* с помощью *VSC* необходимо создать терминал, в котором и будет происходить вся работа с *Git*. Для создания терминала необходимо перейти во вкладку ***Терминал***, которая находится наверху, далее ***Создать терминал***. Экран делится на два окна, верхнее окно - основное поле ввода данных, нижнее - терминал, в котором и будет происходить работа с *Git*.

## Основные команды Git

Существуют некоторые команды, с помощью которых происходит работа с *Git*:
* git init - создание репозитора(хранилище данных, поддерживающий версионность).
* git add - добавляет файл версионности.
* git commit - фиксирует изменения программы.
* git diff - просмотр разницы между версиями.
* git log - просмотр списка всех коммитами.
* git checkout - перемещение между коммитами.

**Важно!!! Все команды пишутся в терминале!!!**

Необходимо отметить тот факт, что программа сама подсказывает какой шаг необходимо сделать пользователю после того, как он её запустит. Эти подсказки всё также прописаны в терминале.

## Как хранить картинки?

Система *Git* создана для хранения разницы между текстовыми данным, поэтому как такового сохранения, например, картинок нет. Но бывает так, что неоходимо зашить в программу какое-либо изображение. Для того, чтобы это сделать необходимо выполнить следующие шаги: 
+ прописать команду в том месте, где необходимо добавить изображение __![текст вместо изображения](название файла с расширением)__.
+ создать новый файл и назвать его так: __.gitignore__
+ внутри нового файла прописать название изображения с расширением.

## Соединение веток

Для соединения веток необходимо выбрать ту ветку с которой необходимо слияние. То есть прописать следующую команду в терминале: **git merge branch_name**, где **branch_name** название ветки, с которой необходимо выполнить слияние.

Бывает так, что при слиянии веток, программа выдаёт ошибку. Не стоит пугаться, что ничего не вышло. Программа не может сделать выбор за пользователя, как именно будет производится редактирование. Пользователь должен сам выбрать один из четырех вариантов, которые предлагает программа.

## Новые команды Git

На последнем занятии были изучены следующие команды:
1. git branch - нужна для отображения всех веток. Чтобы создать новую ветку необходимо прописать *git branch branch_name*.
2. git checkout branch_name - необходима для перемещения между ветками.
3. git merge - выполняет слияние выбранной ветки с той, в которой находится пользователь.
4. Для удаления ветки необходимо прописать следующую команду: *git branch -d branch_name*
5. git log --graph - отображение коммитов по веткам.

## Удалённые репозитории 

Удалённые репозитории нужны для того, чтобы любой член вашей команды мог вносить свои правки в проект и предлагать свой вариант другим участникам проекта. Есть определённая библиотека репозиториев,![которая находится по следующец ссылке](https://github.com/). Ёе принято называть *GitHub*, абсолютно так же как и сайт, на котором хранятся репозитории.

Чтобы работать с удалёнными репозиториями необходимо, для начала, ![зарегестрироваться на сайте](https://github.com/). 

После регестрации вам будет открыт доступ для работы над другими проэктами. 

Для этого необходимо: 

1. Найти нужный репозиторий.
2. *Форкнуть* его, т.е. нажать кнопку *Fork*, после кнопку *Create fork*, т.е. сохранили удалённый репозиторий в свой аккаунт **Github**.
3. В открывшейся вкладке необходимо нажать клавишу *Code*, в открывшемся окне копировать ссылку.
4. Открыть терминал и прописать следующие комманды: 
    + git clone <ссылка> - копирует удалённый репозиторий на локальное устройство
    + cd <Дирректория открывшейся папки после добавления репозитория>

Выполнив все вышеперечисленные шаги, удаленный репозиторий стал локальным репозиторием, в котором можно редактировать программу.

Кроме копирования репозитория, его так же можно выгружать в общее хранилище удалённых репозиториев.

Для этого необходимо: 

1. В терминале прописать следующую команду:
    + git push - направление на выгрузку репозитория
2. Зайти в аккаунт **Github**
3. На странице будет отображено, что пользователь локально редактировал файл и отправил команду на выгрузку.
4. Нажать клавишу *Pull request*

Стоить отметить, что происходит не просто выгрузка локального репозитория в сеть, а слияние локального репозитория с удалённым, т.е. на **Github** выгрузится ваша версия проэкта.
